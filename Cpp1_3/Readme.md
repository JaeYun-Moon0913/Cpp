## debug & Release
디버그(debug) : 오류를 잡는 행위  
릴리즈 : 프로그램을 출시 할 때, 최적화를 하여 필요 없는 코드를 삭제 할 수도 있다.  

코딩을 할 때는 디버그로 실행 하는 것이 좋다.  

## 중단점 
중단점 : 중간에 코드를 실행하여 오류가 있는지 확인하는 기능 , 한줄 한줄 실행 가능 
	- f9 : 중단점 찍기 또는 라인 맨 왼쪽 클릭 하여 빨간색 점 나오게 하기 
	- f10 : 한 줄 실행, 계속 누르면 한 줄 한 줄 씩 실행 가능 
	- 중단점 두개 (시작과 끝) 있을 때는 f5로 시작하면 그 사이가 실행이 된다. 

## 솔루션 탐색기와 팀 탐색기 
솔루션 탐색기 : 솔루션 하위에 여러 프로젝트를 가질 수 있음 
팀 탐색기 창 : git 등을 이용한 버전 관리 가능 

## 빌드 
빌드 : 프로젝트 결과물을 만들기 위해 소스코드 작성 -> 결과물을 만듬(컴파일)->실행파일을 만듬
	- 빌드는 컴파일의 상위 개념으로 볼 수 있다. 
	- 상단 빌드에 Cpp1_3(프로젝트 명) 빌드 : 재 빌드를 함 
	- 상단 빌드에 Cpp1_3(프로젝트 명) 정리 : 디버그 폴더 내용 다 지움
